# AI Mood Tracker

A full-stack mental health mood tracking application with an AI-powered suggestion engine. Log your daily mood, add notes, and get personalized wellbeing tips — all tracked over time with visual stats.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Flutter (Dart) — Web & Android |
| Backend | Node.js + Express |
| Database | PostgreSQL |

---

## Features

- **Mood logging** — Rate your mood 1–5 (Terrible → Amazing) with an optional note
- **AI suggestions** — Rule-based engine generates personalised wellbeing tips based on your mood level and note keywords (stress, anxiety, loneliness, fatigue, etc.)
- **Mood history** — Scrollable list of past entries with timestamps and AI tips
- **Statistics screen** — Total entries, average mood, and positive/neutral/negative breakdown
- **Delete entries** — Swipe or tap to remove any mood entry
- **Dark mode** — Follows system theme automatically
- **Cross-platform** — Runs on Chrome (web) and Android

## Getting Started

### 1. Database

Start PostgreSQL and create the database:

```sql
CREATE DATABASE mood_tracker;
```

The tables are created automatically when the backend starts. Alternatively, run the schema manually:

```bash
psql -U postgres -d mood_tracker -f database/schema.sql
```

### 2. Backend

```bash
cd backend
npm install
```

Create a `.env` file in the `backend/` folder:

```env
DB_USER=postgres
DB_HOST=localhost
DB_NAME=mood_tracker
DB_PASSWORD=your_password
DB_PORT=5432
PORT=5000
```

Start the server:

```bash
# Production
npm start

# Development (auto-reload)
npm run dev
```

The API will be available at `http://localhost:5000`.

### 3. Frontend

```bash
cd frontend
flutter pub get
flutter run -d chrome       # Web (Chrome)
flutter run -d emulator-5554  # Android emulator
```

## API Reference

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/api/health` | Health check |
| `GET` | `/api/moods` | Fetch last 50 mood entries |
| `POST` | `/api/moods` | Log a new mood |
| `GET` | `/api/moods/stats` | Get mood statistics |
| `DELETE` | `/api/moods/:id` | Delete a mood entry |

**POST `/api/moods` body:**
```json
{
  "level": 4,
  "note": "Had a great workout this morning"
}
```

**Response includes an `ai_suggestion` field** generated by the server based on the mood level and note content.

## Environment Notes

- **Chrome (web):** API calls go to `http://localhost:5000`
- **Android Emulator:** API calls go to `http://10.0.2.2:5000` (emulator alias for host localhost)
- **Physical Android device:** Update `baseUrl` in `frontend/lib/services/api_service.dart` to your machine's local IP (e.g. `http://192.168.1.x:5000`)
